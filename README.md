# Biathlon LoadLab — тестово Streamlit приложение

Работещ демонстрационен MVP за дигитално управление на тренировъчното натоварване в биатлона. Приложението използва повторяеми синтетични данни и ръчно въвеждани показатели; няма Strava интеграция.

## Какво е реализирано

- три тестови спортиста с различен профил на поносимост;
- тестови активности и детайлен едносекунден пулсов поток при отваряне на активност;
- реално време `T`, вътрешнозонов коефициент `k`, директен еквивалентен товар `Q` и ефективен товар `E`;
- компонентен индекс 7/40, стабилизиращ базов товар и `Tref`;
- каскада от високи към ниски зони и прагова компонента на разлив;
- експоненциална динамика на остатъчната умора и readiness;
- ръчен дневен мониторинг: сън, умора, стрес, болезненост, болка, сутрешен пулс, HRV и др.;
- ръчни контролни тестове: 3 мин SkiErg, 20 мин около горна Z3 и 200 м спринт срещу наклон;
- интегрирана компонентна готовност и адаптивен множител;
- календар, мезоциклична вълна, акценти, лагери, стартове и тейпър;
- генериране на конкретна 7-дневна програма с автоматично дозирани методи;
- симулатор „Какво ще стане, ако“ с сравнение преди/след;
- роли, треньорски overrides, версии, журнал и DecisionSnapshot;
- CSV/JSON експорти;
- страница „Модели и обяснения“. Основните показатели имат `?` tooltip и линк към подробно обяснение.

## Стартиране

Изисква Python 3.10 или по-нова версия.

### Най-лесно — Windows

Двоен клик върху:

```text
run_demo.bat
```

### Най-лесно — Linux/macOS

```bash
chmod +x run_demo.sh
./run_demo.sh
```

### Ръчно

```bash
python -m venv .venv
# Windows: .venv\Scripts\activate
# Linux/macOS: source .venv/bin/activate
python -m pip install -r requirements.txt
python -m streamlit run app.py
```

След стартиране приложението обикновено се отваря на `http://localhost:8501`.

## Проверка

```bash
pytest -q
```

Тестовете валидират основните формули, обратното преобразуване, мониторинга, тестовите корекции, целия pipeline и зареждането на всички Streamlit страници.

## Работа с демото

1. От „Отборно табло“ избери профил A, B или C.
2. В „Натоварване и 7/40“ отвори активност и сравни реално с еквивалентно време.
3. В „Дневен мониторинг“ промени сутрешните показатели и запиши.
4. В „Контролни тестове“ добави нов валиден резултат.
5. Отвори „Адаптивна програма“, за да видиш новия 7-дневен план.
6. В „Симулатор“ промени активност, мониторинг, тест или дата на основния старт и сравни решението преди/след.
7. Използвай `?` до основните показатели, за да стигнеш до формулите и интерпретацията.

## Данни и ограничения на MVP

Данните се държат в `st.session_state`. Те остават активни в текущата браузърна сесия, но се нулират при рестарт на сървъра. Бутонът „Нулирай всички тестови данни“ връща повторяемия начален сценарий.

Началните коефициенти са експертни параметри за демонстрация и последваща валидация, а не универсални медицински или физиологични норми. Приложението подпомага треньорско решение и не поставя диагнози.

## Структура

```text
app.py                         Streamlit интерфейс и навигация
biathlon/demo_data.py          тестови спортисти, активности, мониторинг, тестове и календар
biathlon/physiology.py         T → k → Q → E, 7/40, Tref, умора, readiness, обратна задача
biathlon/monitoring.py         субективни/обективни показатели и твърди флагове
biathlon/testing.py            контролни тестове и ограничени корекции
biathlon/planning.py           периодизация, седмични цели и конкретен микроцикъл
biathlon/service.py            оркестрация на целия pipeline
biathlon/explanations.py       формули и help текстове
biathlon/charts.py             визуализации
biathlon/constants.py          начални експертни параметри
```

Подробните формули са в [`docs/MODEL_LOGIC.md`](docs/MODEL_LOGIC.md), а нормализираният бъдещ входен договор е в [`docs/DATA_CONTRACT.md`](docs/DATA_CONTRACT.md).
